# 05 Technical Architecture

## Stack

- Next.js 16.2.9.
- React 19.2.4.
- TypeScript.
- Tailwind CSS 4.
- ESLint 9 with `eslint-config-next`.
- npm with `package-lock.json`.

## Runtime

Development environment: Linux.

Observed local versions during documentation sprint:

- Node: `v22.22.2`.
- npm: `10.9.7`.

## App Structure

```text
src/app/
  layout.tsx
  page.tsx
  en/page.tsx
  manifesto/page.tsx
  en/manifesto/page.tsx
  robots.ts
  sitemap.ts

src/components/
  CoreAiChat.tsx
  HomePage.tsx
  ManifestoPage.tsx

public/
  llms.txt
  images/infrastructure/*
```

## Routes

- `/` - Turkish homepage.
- `/en` - English homepage.
- `/manifesto` - Turkish manifesto.
- `/en/manifesto` - English manifesto.
- `/robots.txt` - generated by `src/app/robots.ts`.
- `/sitemap.xml` - generated by `src/app/sitemap.ts`.
- `/llms.txt` - static public file.

## Rendering Model

The app uses the Next.js App Router.

Most page-level components are Server Components by default. `CoreAiChat.tsx` is a Client Component because it uses local React state, refs, effects, and form submission.

## Styling

Global styling is in `src/app/globals.css`.

Tailwind CSS is imported, but the current UI primarily uses custom class names and CSS variables:

- `--background`
- `--surface`
- `--surface-muted`
- `--foreground`
- `--foreground-muted`
- `--accent-gold`
- `--accent-neon`
- `--border-color`

## Integrations

Verified in repo:

- Static Next.js website.
- Static images.
- Metadata, sitemap, robots.
- Deterministic local discovery assistant.
- In-chat contact form with client-side validation.
- Lead draft stored in local component state.

Not verified or not implemented:

- OpenAI or other LLM provider.
- Supabase.
- Telegram.
- WhatsApp.
- Resend.
- Vercel project linking.
- Contact form persistence beyond the active browser session.

## Environment Variables

Current public variable:

- `NEXT_PUBLIC_SITE_URL` - optional production origin override.

No private env vars are required for the current visual prototype.

`.env*` is ignored by `.gitignore`; do not commit secrets.

## Deployment Readiness

The app has the standard scripts needed for a Node-compatible Next.js deployment:

```bash
npm run build
npm run start
```

Before production:

- Run lint and build.
- Verify `NEXT_PUBLIC_SITE_URL`.
- Add OG image if social sharing matters.
- Add global not-found/error states if needed.
- Verify all future integrations in runtime before announcing them.
